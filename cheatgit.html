What is GIT?

  • Git is a version control system for tracking changes in computer files and coordinating work on those files among multiple people.

    • Basic GIT Concepts

  • Repository: The overall package/project
  • Commit: A commit is a change to the repository
  • Branch: An extension of a repository. All repositories, or repos, start with a `master` branch.

    • Commands to Know

  • "cd" (change directory): Changes the directory/folder
  • "cd ~": Will take you to the home directory
  • "cd ..": Moves up one directory
  • "ls": lists the files or directories
  • "git init": Creates a new local repository in the current directory
  • "git clone": Clones a repository from a remote source | Creates a directory inside the current directory, and puts the project in there
    • More Commands...
  • "git status": Tells what files have been added/deleted/modified
  • "git branch .": Stages all changes from the directory
  • "git commit -m": This commits the added changes locally and attaches the commit message to the commit
  • Commit messages should be short but descriptive of all the changes made
  • Committing does not push cnages to the remote repository
